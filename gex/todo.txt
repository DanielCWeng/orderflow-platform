# GEX Morning Snapshot — Claude Code Build Prompt

Build a Python project called `gex-snapshot` that computes and outputs GEX-based key levels every morning before market open. The output should mirror the level structure used by institutional GEX providers: Call Wall, Put Wall, Zero Gamma (gamma flip), Volatility Trigger, Large Gamma 1–4 (ranked by absolute GEX magnitude), and Combo 1–5 (confluence levels across NDX/QQQ/NQ, ranked strongest to weakest).

## Data Layer

### `data/ironbeam.py`

Fetch NQ futures options chain.

**Auth flow:**
POST to `{IRONBEAM_BASE_URL}/auth` with body `{"username": ..., "apikey": ...}`. Extract `token` from response. Use as `Authorization: Bearer {token}` header on all subsequent calls.

**Discover contracts:**
GET `{IRONBEAM_BASE_URL}/info/searchSymbolOptions` with query param `symbol=NQ` (or the active front-month NQ symbol). This returns a list of options contracts. Parse each contract's `exchSym` identifier.

**Get security definitions (strike + type):**
GET `{IRONBEAM_BASE_URL}/info/security/definitions` with `symbols` query param (comma-separated, max 10). Returns `strikePrice`, `optionType` (CALL/PUT), `expirationTime`, `underlyingSymbol` per contract. Batch all discovered NQ option contracts in groups of 10 and call this endpoint iteratively.

**Get quotes (OI + bid/ask):**
GET `{IRONBEAM_BASE_URL}/market/quotes` with `symbols` query param (max 10 per call). From the response, extract per contract:
- `oi` — open interest
- `b` — best bid
- `a` — best ask

Again batch in groups of 10. Merge quote data with security definitions by `exchSym`.

**Return:** A list of dicts, one per contract:
```python
{
  "symbol": str,
  "strike": float,
  "option_type": "CALL" | "PUT",
  "expiration_ts": int,        # unix ms
  "oi": int,
  "bid": float,
  "ask": float,
  "underlying": str            # e.g. "XCME:NQ.M25"
}
```

Filter to front two expiration cycles only to keep computation relevant.

### `data/tradier.py`

Fetch QQQ and NDX equity options chains.

**Auth:** All requests use header `Authorization: Bearer {TRADIER_API_TOKEN}` and `Accept: application/json`.

**Get options chain:**
GET `https://api.tradier.com/v1/markets/options/chains` with params:
- `symbol=QQQ` (then repeat for `NDX`)
- `expiration={nearest_friday}` — compute nearest weekly expiration date dynamically

From response, extract per contract:

- `option_type` ("call" | "put")
- `open_interest`
- `bid`
- `ask`
- `expiration_date`

**Return:** Same dict schema as Ironbeam, normalised:
```python
{
  "symbol": str,
  "strike": float,
  "option_type": "CALL" | "PUT",
  "expiration_date": str,      # YYYY-MM-DD
  "oi": int,
  "bid": float,
  "ask": float,
  "underlying": str            # "QQQ" or "NDX"
}
```

Fetch the two nearest weekly expirations for each ticker (including 0DTE if market is open that day).

## Computation Layer

### `compute/greeks.py`

Implement two pricing functions. Use `scipy.stats.norm` for the normal CDF and PDF.

**Black-Scholes Gamma** (for QQQ, NDX equity options):
```
d1 = (ln(S/K) + (r + 0.5*σ²)*T) / (σ*√T)
Gamma = N'(d1) / (S * σ * √T)
```

**Black-76 Gamma** (for NQ futures options):
```
d1 = (ln(F/K) + 0.5*σ²*T) / (σ*√T)
Gamma = e^(-r*T) * N'(d1) / (F * σ * √T)
where F = current futures price (use mid of bid/ask on the underlying)
```

**IV computation:**
Back-calculate σ from each contract's mid-price ((bid+ask)/2) using Brent's method (`scipy.optimize.brentq`). The objective function is: `BS_price(σ) - market_mid = 0`. Bound σ search between 0.001 and 10.0. If IV cannot be solved (e.g. zero bid/ask, illiquid contract), skip the contract.

Inputs to each function: `S` or `F` (spot/futures price), `K` (strike), `T` (time to expiry in years), `r` (risk-free rate), `σ` (implied vol).

Compute T as: `(expiration_datetime - now_utc).total_seconds() / (365.25 * 24 * 3600)`. Skip any contract where T <= 0.

---

### `compute/gex.py`

**Per-contract GEX:**
```python
sign = +1 if option_type == "CALL" else -1
gex = sign * gamma * oi * 100 * (spot ** 2) * 0.01
```

The `* 100` converts contracts to shares. The `* spot² * 0.01` expresses GEX in dollar terms per 1% spot move.

**Aggregate by strike:**
Sum GEX across all contracts at the same strike price for a given instrument. Return a dict:
```python
{
  strike: float (net GEX at that strike),
  ...
}
```

Do this separately for NQ, QQQ, and NDX. Also compute cumulative GEX (sorted by strike ascending) for gamma flip detection.

## Level Extraction (`levels/extract.py`)

Given the per-strike GEX dict for each instrument, extract the following levels. Do this independently for each instrument first, then combine.

**Per-instrument levels:**
- `call_wall`: strike with the highest positive GEX value
- `put_wall`: strike with the highest negative (most negative) GEX value
- `zero_gamma`: strike where cumulative GEX (sorted ascending by strike) crosses from negative to positive (interpolate between the two surrounding strikes for precision)
- `volatility_trigger`: use the strike nearest to current spot price where net GEX is closest to zero — this is the dealer hedging inflection point
- `large_gamma_1` through `large_gamma_4`: top 4 strikes by absolute GEX magnitude, ranked descending

**Combo level detection (cross-instrument):**
After computing per-strike GEX for all three instruments (NQ, QQQ, NDX), find strike clusters where significant levels from multiple instruments fall within `COMBO_TOLERANCE_POINTS` of each other.

Algorithm:
1. Collect all `large_gamma_1` through `large_gamma_4` strikes from each of the three instruments — giving up to 12 candidate strikes total.
2. For each candidate strike, check how many other instrument strikes fall within `±COMBO_TOLERANCE_POINTS`.
3. Score each cluster: sum of absolute GEX magnitudes from all contributing strikes across instruments.
4. Deduplicate overlapping clusters (merge if cluster centres within `COMBO_TOLERANCE_POINTS/2`).
5. Rank clusters by score descending → `combo_1` (strongest) through `combo_5` (weakest). Discard any cluster where only one instrument contributes — true combos require at least two instruments.

Attach to each combo level: which instruments contributed, the individual strikes, and the weighted average strike price as the reported level.

**Return a single structured dict:**
```python
{
  "nq":  { "call_wall": ..., "put_wall": ..., "zero_gamma": ..., "volatility_trigger": ..., "large_gamma": [lv1, lv2, lv3, lv4] },
  "qqq": { ... },
  "ndx": { ... },
  "combo": [
    { "rank": 1, "strike": ..., "instruments": ["NQ", "QQQ"], "score": ... },
    ...
  ],
  "spot": { "nq": ..., "qqq": ..., "ndx": ... },
  "generated_at": "2025-06-03T08:45:00Z"
}
```

2. Fetch QQQ options chain from Tradier
3. Fetch NDX options chain from Tradier
4. Compute IV and Gamma for each contract across all three instruments

6. Extract levels
7. Output

## Notes for implementation

- NQ strikes are in index points. QQQ is in dollars. NDX is in index points. The combo level tolerance (`COMBO_TOLERANCE_POINTS`) should be applied in the native units of each instrument but normalised conceptually — NQ and NDX are roughly 40× QQQ, so the tolerance for QQQ-vs-NQ comparisons should scale accordingly. Implement this scaling in the combo detection logic.
- For the Ironbeam `searchSymbolOptions` call, filter to options where `expirationTime` is within the next 30 days to avoid pulling the entire options universe.
- Get current NQ spot price from Ironbeam's `/market/quotes` endpoint on the front-month futures symbol. Get QQQ and NDX spot from Tradier's `/v1/markets/quotes` endpoint.
- Use concurrent.futures.ThreadPoolExecutor to parallelise the three Ironbeam batched quote fetches (NQ chain is large). Keep Tradier calls sequential to avoid rate limiting.
- Target runtime under 60 seconds for the full pipeline.
